# 通用 Skill 工作流指导

适用范围：
- 没有明确行业属性的通用任务。
- 横跨多个行业的知识整理、任务拆解、沟通协作、报告生成和信息核验。

工作流抽取建议：
- 优先从 SKILL.md 的 Purpose、Workflow、Steps、Constraints、Output 等部分提取真实业务步骤。
- 每个步骤应能回答“这一步为什么存在、输入是什么、输出是什么、完成条件是什么”。
- 避免把“理解用户输入”“调用模型”“输出结果”当作全部工作流；这些是运行脚手架，不是业务步骤。

命名与输出质量：
- node.id 使用 snake_case，并体现业务语义。
- source_evidence 必须能追溯到 SKILL.md 或 README.md。
- 最终输出应清晰说明结论、依据、未确认事项和建议动作。
